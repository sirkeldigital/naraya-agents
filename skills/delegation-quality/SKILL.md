# Skill: Delegation Quality

Use this skill when dispatching sub-agents or reviewing delegated output.

## Delegation Rules
- Delegate only when it improves speed, independence, or expertise.
- Give atomic tasks with explicit deliverables, tools, constraints, and expected evidence.
- Parallelize independent work instead of serial delegation.
- Do not duplicate delegated searches unless output is incomplete or contradictory.

## Acceptance Criteria
- Implementation delegations must return Summary, Files, Verification, and Risks.
- Research delegations must return Summary, Evidence, Sources, Risks, and Recommended next step.
- Weak evidence is not verified evidence.
- Missing sections require follow-up or manual completion before final claims.
