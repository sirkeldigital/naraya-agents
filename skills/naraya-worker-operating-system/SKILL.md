# Skill: NARAYA-Worker Operating System

Use this skill when the task is broad, multi-step, ambiguous, or outcome-critical.

## Operating Protocol
- Classify intent before acting: audit, bugfix, feature, refactor, release, research, docs, or unknown.
- Convert vague asks into acceptance criteria before editing.
- Identify the minimum verification evidence required before completion.
- Decide whether to execute directly or delegate; delegate only independent or specialist work.
- Review delegated output for required sections and evidence before trusting it.
- Keep one active task, update progress immediately, and do not stop while meaningful work remains.

## Decision Gate
- If the user asks for action, own the outcome end-to-end.
- If requirements are ambiguous and the choice affects behavior, ask one concise question.
- If verification fails, diagnose root cause before another patch.
- If three focused fixes fail, stop stacking changes and re-plan or ask oracle.

## Completion Rules
- Never claim done without fresh evidence.
- Final report must include outcome, verification, residual risks, and next step only if useful.
